## 导出对象名称的变化

您可能需要在运行时访问某些对象，例如通过文本运行（text run）来更换字符串，或者访问组件的输入。为了使这些对象在运行时可被搜索到，您需要显式设置将其名称导出。

以前，如果在编辑器中将对象的名称从默认值更改，那么名称就会被导出。这种方式的问题在于，它假设任何重命名的对象都需要在运行时被查找，但在很多情况下，您可能仅仅是为了更好地组织文件而重命名对象，并不一定需要将它们导出到 `.riv` 文件中。因此，我们更改了这种方式，以便更精确地控制哪些对象的名称会被导出。

要导出名称，请在层级结构或舞台上右键点击该对象，并勾选 "Export name"（导出名称）选项。

::: info
名称设置为导出的对象，可以通过其名称周围包裹的方括号来识别。
:::

![Image](/images/editor/exporting/8a147f5b-4e93-4d45-8984-64746ae1417d.webp)

::: info
动画、状态机、事件和输入的名称不需要手动设置导出。
:::

### 优化名称的好处

将对象的名称导出到 `.riv` 文件会增加少量数据。对于大型、复杂的文件，名称数据累加起来也是一笔开销。因此，只导出运行时需要引用的名称是更理想的做法。

### 在显式导出功能推出之前创建的文件

对于在此功能上线之前创建的文件，我们假设任何重命名的对象在运行时都是需要可搜索的。这意味着当您打开现有的老文件时，可能会注意到层级结构中有很多项都显示在方括号内。如果您希望不再导出这些名称以减小导出文件的大小，可以按照以下步骤操作：

1. 从工具栏菜单中选择 `Export options` > `Remove name exports`（导出选项 > 移除名称导出）。
2. 在层级结构或舞台上右键点击运行时需要访问的对象，并选择 "Export name"，逐个重新启用名称导出。
