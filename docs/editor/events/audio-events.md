"音频事件"（Audio Events）提供了在动画中和/或在响应用户交互时触发声音效果的方法。就像现有的事件一样，它们可以由时间轴上的关键帧、状态过渡期间或通过监听器触发。

音频事件代表了 Rive 音频功能的第一阶段——它们提供了一种触发可以相互叠加的声音效果的理想方式。

以前，要使用事件触发音频，需要使用 Rive 运行时之一并在您的应用程序或游戏中进行一些开发工作。在 Rive 编辑器中直接引入音频事件简化了为动画添加声音的过程——进一步赋能设计师，同时简化了开发者的实现。

::: info
音频事件非常适合触发短促的声音，以响应用户交互或配合角色动画。虽然由于长音频（如背景音乐和配音）也可以通过音频事件触发，但它们缺乏随时间操纵音量、声像（panning）等的能力。请关注即将推出的其他音频功能，它们将为其他音频用例提供更佳的解决方案。
:::

## 音频资产 (Audio assets)

### 导入音频

将音频资产直接拖放到 Rive 文件中，将其添加到资产面板。或者，您可以通过层级结构顶部的选项卡导航到资产面板。选择音频类别旁边的"添加"操作，以调出操作系统的文件浏览器。

::: info
Soundly 提供的集成声音库对所有人开放。但是，上传自定义音频文件仅限 Pro 用户。
:::

### Soundly

我们与 Soundly 合作，让每个人都能直接访问其免费库中由音频专业人士制作的 3,000 多种声音。在工具栏汉堡菜单中找到 "Browse Sounds"（浏览声音）选项即可查看 Soundly 的免费库。

选择"播放"操作或点击波形图可以在整个声音面板中预览声音。选择"添加"操作将选定的声音移至资产面板，以便在 Rive 文件中使用。

### 创建剪辑 (Clips)

您可能想从较长的源音频资产中创建一个较短的剪辑，特别是当单个文件中包含多个音效时。要创建剪辑，首先在资产面板中选择源音频文件。选中所选资产后，波形面板将出现在舞台底部。使用扩展器显示波形。

要创建新剪辑，请在波形上点击并拖拽以突出显示一个范围。可以通过灰色边界调整开始和结束点。对剪辑范围满意后，选择"添加"操作保存剪辑。它将显示在资产面板中原始音频源的下方。

::: info
剪辑会在导出时生成新的音频文件。
:::

通过选择源音频资产并重复该过程，可以创建额外的剪辑。波形剪辑器包含一个下拉菜单，可在现有剪辑之间切换，并根据需要进行调整。

### 设置音量

选择音频资产或相应的剪辑，在检查器中设置其音量。请注意，音量目前还不能随时间设置关键帧。即将推出的音频功能（包括"音频组"和"音频发射器"）将引入设置音量、声像等关键帧的选项。

### 导出选项

与其他资产格式类似，您可以配置音频资产的导出选项以优化运行时性能。

**导出类型：** 在 **Assets**（资产）面板中选择音频文件，并更改 **Type**（类型）选项，以定义您希望音频文件导出到何处。

![Image](/images/editor/events/a4ebb837-e3fa-4d3d-a730-39faf4914047.webp)

- **内嵌 (Embedded)：** 将音频文件嵌入 `.riv` 文件中。这是最简单的选项，但会增加 Rive 文件的大小。
- **引用 (Referenced)：** 将音频文件与 `.riv` 文件并排导出。如果您有多个使用相同音频资产的 Rive 文件，或者您想在运行时更改音频资产，此选项非常理想。使用引用音频文件将减小 Rive 文件的大小。
- **托管 (Hosted)：** 托管音频文件上传到 Rive 的 CDN，以便运行时按需下载。与引用类似，选择在 Rive 的 CDN 上托管资产将使其从 `.riv` 中移除，从而减小导出的文件大小。当您的动画在应用、游戏或网站中播放时，Rive 运行时将获取音频。

::: info
托管在 Rive CDN 上的资产任何拥有链接的人都可以访问。
:::

::: info
托管资产在 Voyager 和 Enterprise 方案中可用。[了解更多关于我们的方案和定价](https://rive.app/pricing)。
:::

**格式：** 选择将您的音频文件导出为 `.wav`、`.flac` 或 `.mp3`。

**质量：** 当将音频导出为 mp3 文件时，您还可以通过质量字段设置压缩级别。

## 创建音频事件

创建音频事件最简单的方法是将音频资产或剪辑从资产面板直接拖动到舞台上。这样做会通过预分配的资产创建一个事件。或者，通过激活事件工具（`SHIFT + E`）并点击舞台来创建一个普通事件。创建后，在检查器中将类型设置更改为 "Audio"。对于音频事件，将显示分配资产和浏览 Soundly 库的额外选项。

## 触发音频事件

与普通事件一样，音频事件可以通过几种不同的方式触发：

- **时间轴 (Timeline)：** 在动画模式下，选中时间轴后，事件检查器将出现一个设置事件关键帧的按钮。为事件设置关键帧会导致该事件被报告。在音频事件的上下文中，报告它将开始播放分配的音频资产。
- **过渡 (Transitions)：** 在状态机内选择一个过渡节点，并通过检查器添加一个事件。您可以选择事件应在过渡开始还是结束时报告。
- **监听器 (Listeners)：** 在状态机内选择一个监听器，并通过检查器添加一个 "Report Event"（报告事件）动作。指针选项将决定音频何时播放。例如，将指向形状的"按下"（Pointer Down）监听器分配一个音频事件，将在用户点击该形状时开始播放。

### 监测音频

音频电平可以通过检查器底部的 VU 表进行监测。使用 VU 表检查是否出现爆音（clipping）。如果多个音频事件同时播放，可能会导致总输出爆音。如果您注意到峰值电平变红，请考虑降低音频资产的音量，以留出更多余量。
